# Video Annotation Tool - 実装完了報告

野球の試合映像にタグ付けを行うための Video Annotation Tool のフロントエンド実装が完了しました。

## 実装された機能

### 1. 動的タグカテゴリ管理 & JSONインポート (Updated!)
「球種」「結果」以外にも、任意のカテゴリ（**カウント、判定、野手守備など**）を自由に追加・編集できるようになりました。
- **設定モーダル:** 歯車ボタンからカテゴリの追加やタグプリセットの管理ができます。
- **JSONインポート:** 用意した `tags.json` を設定画面からアップロードすることで、カテゴリとタグを一括設定できます。
- **コンテキストメニュー:** 右クリックメニューに設定した全カテゴリが自動で表示されます。

### 2. タイムラインと栞（ブックマーク）
- **2段階タイムライン:** 全体を見渡す「Overview」と、前後30秒を詳細に表示する「Detail (Zoom)」を搭載。
- **栞の挿入 (Add):** `B` キーで現在位置に挿入できます。
- **栞の削除 (Delete):** `Shift + B` キー、または右クリックメニューから、現在位置の栞を削除できます。削除した栞は `Ctrl + Z`（または `Cmd + Z`）で元に戻せます。
- **視認性向上:** Overview上にも赤い線で全ブックマークが表示されるようになりました。

### 3. 操作性とサイドバー
- **検出感度の調整:** 撮影停止によるカット検出の感度を調整できます（**デフォルト値を23.0に設定しました**）。調整後は「再解析」ボタンで反映されます。
- **設定の永続化:** 検出感度や作成したタグカテゴリはブラウザに保存され、次回起動時も維持されます。
- **サイドバー設定:** 頻繁に使う感度調整と再解析ボタンはサイドバーに常駐しています。

### 4. ロリポップ！マネージドクラウドへのデプロイ & 最適化 (New!)
バックエンドをロリポップ！マネージドクラウドにデプロイし、環境特有の制限を克服して安定稼働を実現しました。
- **チャンクアップロード方式の刷新:** 
    - Nginxの1MB制限やWAFによる `multipart/form-data` の遮断を回避するため、**Base64エンコードされたJSON** でデータを細切れに送る方式を実装しました。
    - これにより、100MBを超える大きな動画ファイルも、ロリポップのプロキシに弾かれることなくアップロード・解析が可能です。
- **ストレージ最適化:** 
    - サーバーの `/tmp` (256MB RAM-disk) の容量不足によるエラーを回避するため、一時ファイルの保存先をディスク領域（`/var/app/current/uploads/`）に変更しました。
- **進捗フィードバックの向上:** 
    - プログレスバーに「アップロード中...」と「解析中...」のステータスを表示するようにし、長時間かかる処理でもユーザーが進行状況を把握できるように改善しました。

## 実行方法
プロジェクトディレクトリにて開発サーバーまたはビルド後の資材が利用可能です。

### ローカル開発環境 (Lolipopバックエンド接続)
```bash
cd frontend
npm run dev
# 歯車アイコンから Backend URL を https://romantic-hirado-5305.lolipop.io/ に設定
```

## 課題解決のまとめ
- [x] **500 Internal Server Error (Uvicorn not found):** 起動コマンドを `python -m uvicorn ...` に修正して解決。
- [x] **CORS / 413 Payload Too Large:** チャンクサイズを縮小し、Base64 JSON方式に切り替えて解決。
- [x] **No space left on device:** `/tmp` ではなくディスク上の `/uploads` ディレクトリを使用するように変更して解決。
- [x] **解析状況が0%で固まる:** アップロード進捗をUIに反映し、チャンクサイズを1MBに最適化することで視認性と速度の両方を向上。
