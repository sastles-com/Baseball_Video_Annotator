# Video Analyzer V2 Performance & UI Optimization

ビデオ解析システムのパフォーマンスを根本から改善し、V2アーキテクチャへと移行しました。

## 実装された主な改善点

### 1. 超高速バイナリ通信 (Binary Chunking)
- **Base64変換の廃止**: 従来の JSON による Base64 送信を止め、生のバイナリデータ（ArrayBuffer）を直接 `FormData` で送信するように変更しました。
- **データ量の削減**: 通信データ量が約 30% 低減し、ブラウザの CPU 負荷も劇的に下がりました。
- **チャンクサイズの最適化**: 1MB から 5MB に拡大し、通信リクエスト回数を 1/5 に削減。

### 2. 解析スピードの劇的向上 (Frame Resizing)
- **バックエンドでのリサイズ**: OpenCV が処理を行う前に、フレームを解析に十分なサイズ（320px）にダウンスケーリングします。
- **計算時間の短縮**: CPU への負荷を最小限に抑え、解析完了までの時間を数倍高速化しました。

### 3. UI レスポンスとビデオ操作の正常化
- **再描画の最適化 (React Optimization)**: 
  - `App.tsx` から高頻度な `played` 状態の購読を削除しました。
  - プログレスバーや感度設定を独立したコンポーネントに分離。
  - これにより、動画再生中にアプリ全体が毎秒 60 回再描画されるボトルネックを解消しました。
- **Timeline のメモ化**: 
  - 多数のブックマークがある場合でも、背景グリッドやマーカーの描画負荷を抑えるようメモ化を徹底しました。
- **シーク・ホイール操作の復旧**:
  - ホイールによるコマ送りのレスポンスを改善し、シークバー操作時の 20-30 秒におよぶフリーズを解消しました。

## 検証結果
- **アップロード**: 100MB 規模の動画が数秒〜十数秒で完了。
- **解析**: リサイズ処理により、以前より圧倒的に早くブックマークが生成される。
- **操作性**: 解析完了後（数百のブックマークがある状態）でも、シークやホイール操作が遅延なく行える。

---

> [!TIP]
> **操作のコツ**:
> - **マウスホイール**: ビデオプレイヤー上でホイールを動かすと、1フレームずつ正確に移動できます（コンソールに `Video: wheel step` と表示されます）。
> - **シークバー**: 動画の長さに関わらず、現在は瞬時に反映されるはずです。
