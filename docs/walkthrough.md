# Video Annotation Tool - 実装完了報告

野球の試合映像にタグ付けを行うための Video Annotation Tool の実装と最適化が完了しました。

## 実装された機能

### 1. 動的タグカテゴリ管理 & JSONインポート
「球種」「結果」以外にも、任意のカテゴリ（**カウント、判定、野手守備など**）を自由に追加・編集できるようになりました。
- **設定モーダル:** 歯車ボタンからカテゴリの追加やタグプリセットの管理ができます。
- **JSONインポート:** 用意した `tags.json` を設定画面からアップロードすることで、カテゴリとタグを一括設定できます。

### 2. ビデオ再生とシークの安定化 (Hotfix!)
シークバーの操作に動画が追従しない問題を徹底的に修正しました。
- **メモ化による描画分離:** 進捗バーの更新がビデオ再生に干渉しないよう、ビデオ要素を独立して保護しました。
- **高精度同期:** Reactの状態とネイティブVideo要素の状態をより精密に同期させ、快適なブラウジングを実現。
- **コマ送り機能:** マウスホイールによる直感的なコマ送りが可能です。

### 3. パフォーマンス最適化 (Update!)
大容量動画のアップロード速度と、バックエンドの視認性を改善しました。
- **高速アップロード:** メインスレッドの占有時間を最適化し、初期実装に近い高速なチャンク送信を再開しました（音声の途切れも抑制）。
- **ログの静音化:** 
    - `/api/health` のログを完全に非表示。
    - `/api/upload-chunk` のログを **100回に1回** に制限。
    - これにより、バックエンドのコンソールに重要な情報（解析開始、エラーなど）が埋没することを防いでいます。

### 4. ロリポップ！マネージドクラウドへのデプロイ
- **Base64 JSON方式:** ロリポップのNginx制限を回避し、大容量ファイルの送信を実現。
- **ディスク共有ストレージ:** `/tmp` の容量制限（256MB）を回避し、数GBクラスの動画も扱えるようになっています。

## 実行方法

### ローカル開発環境 (Lolipopバックエンド接続)
```bash
cd frontend
npm run dev
# 歯車アイコンから Backend URL を Lolipop の URL に設定
```

### アーキテクチャのまとめ
- **Frontend:** React + Vite + Zustand + Tailwind CSS
- **Backend:** FastAPI + OpenCV (Headless) + Uvicorn
- **Infrastructure:** Lolipop Managed Cloud (Docker-like environment)
